# ระบบ Cache สำหรับ PostgreSQL Data Viewer

## ฟีเจอร์ใหม่ที่เพิ่มเข้ามา

### 1. ระบบ Cache อัตโนมัติ
- เมื่อดึงข้อมูลจาก PostgreSQL สำเร็จ โปรแกรมจะเก็บผลลัพธ์ไว้ในฐานข้อมูล SQLite ชั่วคราว (item_cache.db)
- การดึงข้อมูลครั้งต่อไปจาก Query เดียวกันจะใช้ข้อมูลจาก Cache แทนการเชื่อมต่อ PostgreSQL

### 2. ปุ่มและฟังก์ชันใหม่

#### ปุ่ม "🔄 รีเฟรช Cache"
- บังคับดึงข้อมูลใหม่จาก PostgreSQL โดยข้าม Cache
- ใช้เมื่อต้องการข้อมูลล่าสุดจากฐานข้อมูล

#### ปุ่ม "⚙️ ตั้งค่า Cache"
เปิดหน้าต่างตั้งค่า Cache ที่มีฟังก์ชัน:
- **ปรับเวลาหมดอายุ Cache**: เปลี่ยนระยะเวลาที่ Cache จะหมดอายุ (ค่าเริ่มต้น 30 นาที)
- **ดูสถานะ Cache**: แสดงจำนวน Query ที่มีใน Cache และจำนวนที่ยังไม่หมดอายุ
- **ลบ Cache ทั้งหมด**: ลบข้อมูล Cache ทั้งหมดเพื่อเริ่มต้นใหม่

### 3. แสดงสถานะ Cache
- แสดงข้อความ "จาก Cache ✔" เมื่อดึงข้อมูลจาก Cache
- แสดงข้อความ "สำเร็จ ✔" เมื่อดึงข้อมูลจาก PostgreSQL
- แสดงสถิติ Cache: "Cache: X/Y queries (หมดอายุใน Z นาที)"

## วิธีการทำงานของ Cache

1. **การบันทึก Cache**: ใช้ Hash ของ SQL Query + ข้อมูลการเชื่อมต่อเป็น Key
2. **การตรวจสอบ Cache**: ตรวจสอบว่ามีข้อมูลใน Cache และยังไม่หมดอายุหรือไม่
3. **การหมดอายุ**: Cache จะหมดอายุตามเวลาที่ตั้งค่าไว้ (ค่าเริ่มต้น 30 นาที)

## ประโยชน์ของระบบ Cache

1. **ลดเวลาในการรอ**: ข้อมูลที่เคยดึงแล้วจะแสดงผลทันทีจาก Cache
2. **ประหยัดทรัพยากรเครือข่าย**: ลดการเชื่อมต่อฐานข้อมูลที่ไม่จำเป็น
3. **เพิ่มความเสถียร**: ลดความเสี่ยงจากปัญหาเครือข่ายหรือฐานข้อมูลไม่พร้อมใช้งาน
4. **ใช้งานร่วมกัน**: Cache สามารถใช้งานร่วมกันระหว่างการเปิดโปรแกรมหลายครั้ง

## ไฟล์ที่เกิดขึ้น

- `item_cache.db`: ไฟล์ฐานข้อมูล SQLite ที่เก็บ Cache (จะสร้างอัตโนมัติ)

## หมายเหตุ

- การเปลี่ยนแปลง SQL Query หรือข้อมูลการเชื่อมต่อจะทำให้สร้าง Cache ใหม่
- Cache จะไม่ส่งผลต่อการทำงานของ Excel Tab
- สามารถลบไฟล์ `item_cache.db` เพื่อเริ่มต้น Cache ใหม่ได้
